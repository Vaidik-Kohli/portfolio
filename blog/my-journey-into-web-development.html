My Journey Into Web Development

Web Dev Journey Learning

Everyone has a different story of how they got into coding. Mine started with curiosity, a lot of YouTube tutorials, and an old laptop that could barely run VS Code. Here's how it all began.

The Spark

It all started when I was 14. I saw a friend build a simple website using HTML and CSS, and I was fascinated. The idea that you could type some text into a file and see something appear in a browser felt like magic. I went home that day and googled "how to make a website", and I've been hooked ever since.

My first HTML page was hilariously bad — a bright yellow background with Comic Sans text and a bunch of <br> tags instead of proper CSS. But it was mine, and I was proud of it.

Finding My Path

After the initial excitement of HTML/CSS, I dove into JavaScript. This is where things got real. Suddenly I wasn't just placing elements on a page — I was making things happen. Click handlers, DOM manipulation, simple animations — each new concept felt like unlocking a superpower.

I started with free resources:

  • freeCodeCamp — The structured curriculum helped me build a foundation
  • YouTube channels — Fireship, Traversy Media, and Kevin Powell became my virtual teachers
  • MDN Web Docs — When I needed to really understand how something worked
  • Building projects — This is where the real learning happened

The Design Side

Around the same time, I discovered Figma. As someone who always appreciated aesthetics, design tools felt natural to me. I started experimenting with UI layouts, learned about color theory, typography hierarchies, and spacing systems.

This dual interest in both design and development turned out to be my biggest strength. While many developers struggle with making things look good, and many designers struggle with code, I found myself comfortable in both worlds.

"The best way to learn to code is to code. The best way to learn design is to design. There are no shortcuts — only consistent practice."

Where I Am Now

Today, I'm 16 and actively freelancing as a designer and developer. I work with Python, JavaScript, React, and Node.js on the development side, and Figma, Photoshop, and Lightroom on the design side.

I'm also preparing for BITS Pilani Goa (CSE 2028), which means balancing academics with my passion for building things. It's challenging, but I wouldn't have it any other way.

What's Next

I want to dive deeper into full-stack development — specifically building real products that solve problems. I'm excited about learning databases, APIs, authentication, and deployment. The goal is to go from "I can build a website" to "I can build a product."

If you're reading this and you're just starting out — my advice is simple: start building. Don't wait until you feel ready. You never will. Just pick a project, struggle through it, google everything, and finish it. That's how you learn.
